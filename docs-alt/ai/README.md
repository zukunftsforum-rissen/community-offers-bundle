# AI Onboarding Documentation

This directory contains documentation intended for **AI assistants working on the project**.

Purpose:

- Provide a fast conceptual overview of the system
- Help an AI navigate the codebase
- Explain domain concepts and workflows
- Point to important services, controllers, and tables

⚠️ Important:

The **codebase is the canonical source of truth**.

This documentation is intentionally simplified and may lag behind the implementation.
Whenever precise details are needed (API parameters, database fields, state transitions),
the AI must verify them against the actual code.

Recommended reading order for a new AI:

1. architecture.md
2. domain-model.md
3. api-flows.md
4. glossary.md
5. generated/class-index.md
